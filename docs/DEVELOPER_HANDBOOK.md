# Developer Handbook

## Welcome

This is a beginner-friendly, professional team project. Small, focused changes are easier to review, test, and merge.

## Daily checklist

1. Pull the latest `main` before starting work.
2. Work only on your assigned Jira task and branch.
3. Check the page at desktop and mobile widths.
4. Use semantic HTML, readable class names, and the shared CSS variables.
5. Open a Pull Request when the task is ready for review.

## Code standards

- Use semantic elements such as `header`, `nav`, `main`, `section`, and `footer`.
- Keep HTML, CSS, and JavaScript in their respective folders.
- Prefer reusable classes and CSS variables over repeated styles.
- Add meaningful `alt` text to informative images.
- Keep JavaScript small, named, and scoped to the feature it supports.
- Do not add frameworks, packages, or build tools without team approval.

## Before you request review

Verify links, keyboard navigation, images, responsive layout, browser console errors, and the task acceptance criteria.
