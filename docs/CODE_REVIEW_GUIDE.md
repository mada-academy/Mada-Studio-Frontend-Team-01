# Code Review Guide

## For authors

- Keep each Pull Request focused and explain the reason for the change.
- Include screenshots or recordings for visible UI changes.
- Test the task before requesting review.
- Respond to feedback constructively and resolve conversations only after acting on them.

## For reviewers

Review for the task requirements, responsive behavior, semantic HTML, accessibility basics, maintainability, and unintended changes. Be specific, kind, and actionable.

## Approval standard

Approve only when the change meets the Jira acceptance criteria, works in the browser, and has no unresolved required comments. Suggestions that are not blockers should be clearly labeled as optional.
