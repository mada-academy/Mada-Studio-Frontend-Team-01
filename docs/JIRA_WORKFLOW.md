# Jira Workflow

## Task lifecycle

1. **To Do** — the task is defined and ready to be picked up.
2. **In Progress** — a student has started work and created a branch.
3. **In Review** — a Pull Request is open and linked to the task.
4. **Done** — the Pull Request is approved and merged into `main`.

## Working with a task

- Read the description and acceptance criteria before coding.
- Assign the task to yourself and move it to **In Progress**.
- Include the task key in the branch or commit when your team uses Jira automation, for example `feature/MADA-12-project-cards`.
- Paste the Jira task URL in the Pull Request template.
- Move the task to **In Review** when the PR opens; mark it **Done** after merge.

If a task grows too large or its requirements are unclear, ask the team lead before expanding its scope.
